## 1. 产品概述

### 1.1 一句话描述
让用户在浏览网页时，通过浏览器原生收藏或插件一键收藏，自动把网页存入 bookmark-lite 平台的个人书签库。

### 1.2 目标用户
已注册 bookmark-lite 的个人用户（`USER` 数据域）。

### 1.3 价值
- 收藏链路从「切回平台手动录入」缩短为「原生收藏即同步」或「点一下即存」。
- 不改变用户既有的浏览器收藏习惯。

### 1.4 名词定义
| 术语 | 含义 |
|---|---|
| 被动同步 | 用户用浏览器原生方式收藏，插件监听后自动推送平台 |
| 主动收藏 | 用户在插件 popup 点「收藏当前页」 |
| 同步开关 | 控制被动同步是否生效，默认开启 |
| API Token | 平台为用户签发的访问令牌，插件凭此调用平台接口 |
| 未分类 | 平台中无标签的书签视图（`untagged`） |
| 去重 | 同一用户下相同规范化 URL 视为同一书签 |

## 2. 功能范围

### 2.1 范围内（v1）
1. 平台设置页生成 / 查看 / 撤销 API Token。
2. 插件 popup 配置：填写 Token、同步开关（默认开）。
3. 被动同步：原生新增书签 → 推送平台。
4. 主动收藏：popup 一键收藏当前页（不编辑）。
5. 去重：已存在则跳过，主动收藏时提示「已在书签库」。
6. 失败重试：推送失败入队重试，彻底失败角标提示。

### 2.2 范围外（v1 不做）
- 删除 / 修改同步、反向同步、双向同步。
- 历史书签全量导入。
- 同步时自动打标签（统一进未分类）。
- 收藏时编辑标题 / 选标签。
- Firefox / Safari。

## 3. 用户流程

### 3.1 首次配置
```
用户登录平台 → 设置页 → 生成 API Token → 复制明文
→ 安装插件 → 打开 popup → 粘贴 Token → 保存
→ 同步开关默认已开启
```

### 3.2 被动同步
```
用户在任意网页按 Ctrl+D / 点星标完成原生收藏
→ 插件监听到新增（同步开关为开）
→ 自动推送平台个人书签库（无打扰，状态可在 popup 查看）
```

### 3.3 主动收藏
```
用户浏览网页 → 打开插件 popup → 点「收藏当前页」
→ 即时存入平台 → 提示「已收藏」/「已在书签库」
```

## 4. 详细需求

### 4.1 API Token 管理（平台设置页）
| 编号 | 需求 |
|---|---|
| RQ-T1 | 用户可输入名称生成新 Token，明文仅在生成时展示一次，提供一键复制 |
| RQ-T2 | 关闭明文弹窗后不可再次查看完整 Token |
| RQ-T3 | Token 列表展示名称、前缀（如 `blt_xxxx***`）、最后使用时间、创建时间 |
| RQ-T4 | 用户可撤销任一 Token，撤销后立即失效 |
| RQ-T5 | 单用户可拥有多个 Token |
| RQ-T6 | Token 数据库仅存哈希，不存明文 |

### 4.2 插件配置（popup）
| 编号 | 需求 |
|---|---|
| RQ-C1 | popup 提供 Token 输入框与保存 |
| RQ-C2 | popup 提供同步开关，默认开启 |
| RQ-C3 | 未配置 Token 时，popup 显著提示需先配置 |
| RQ-C4 | popup 显示当前状态（已连接 / Token 无效 / 待重试数量） |

### 4.3 被动同步
| 编号 | 需求 |
|---|---|
| RQ-P1 | 同步开关为开时，原生新增书签触发推送 |
| RQ-P2 | 同步开关为关时，不推送 |
| RQ-P3 | 新增的是文件夹（无 URL）时跳过 |
| RQ-P4 | 仅处理开关开启后的新增，不导入历史书签 |
| RQ-P5 | 推送的书签 scope 为 `USER`，进入未分类 |

### 4.4 主动收藏
| 编号 | 需求 |
|---|---|
| RQ-A1 | popup「收藏当前页」一键存入，不进入编辑 |
| RQ-A2 | 自动取当前标签页 URL / 标题 / favicon |
| RQ-A3 | 成功提示「已收藏」 |
| RQ-A4 | 命中去重提示「已在书签库」，不报错、不重复创建 |

### 4.5 去重与一致性
| 编号 | 需求 |
|---|---|
| RQ-D1 | 同一用户下按规范化 URL 去重 |
| RQ-D2 | 已存在则跳过（静默成功），不覆盖既有标题 / 标签 |
| RQ-D3 | 主动与被动两条路径行为一致，走同一后端接口 |

### 4.6 失败处理
| 编号 | 需求 |
|---|---|
| RQ-F1 | 网络 / 5xx 失败时，载荷入本地队列 |
| RQ-F2 | service worker 启动、下次书签事件、popup 打开时尝试重试 |
| RQ-F3 | 每条最多重试 3 次（指数退避），超限标记失败 |
| RQ-F4 | 存在失败项时，扩展图标显示角标数量 |
| RQ-F5 | 401（Token 无效）不重试，提示用户检查 Token |

## 5. 接口契约（验收基线）

`POST /api/extension/bookmarks`，请求头 `Authorization: Bearer <token>`。

| 场景 | HTTP | 响应 data/error |
|---|---|---|
| 新建成功 | 200 | `{ ok:true, data:{ id, alreadyExists:false } }` |
| 已存在去重 | 200 | `{ ok:true, data:{ alreadyExists:true } }` |
| 缺/错 Token | 401 | `{ ok:false, error:{ code:"AUTH_REQUIRED" } }` |
| 参数非法 | 422 | `{ ok:false, error:{ code:"VALIDATION_FAILED", fieldErrors } }` |
| 服务异常 | 500 | `{ ok:false, error:{ code:"INTERNAL_ERROR" } }` |
| 预检 | 204 | CORS 头：Allow-Origin / Headers / Methods |

## 6. 验收用例（Acceptance Test Cases）

> 标注：✅ 通过条件。每条用例需可独立执行、可复现。

### 6.1 Token 管理
| 用例 | 步骤 | 预期 |
|---|---|---|
| AC-T1 | 设置页输入名称「我的Chrome」生成 Token | ✅ 弹出完整明文，可复制；列表新增一条 |
| AC-T2 | 关闭明文弹窗后刷新页面查看列表 | ✅ 仅显示前缀，无完整明文 |
| AC-T3 | 点击某 Token 撤销 | ✅ 列表移除/标记撤销，该 Token 立即调用接口返回 401 |
| AC-T4 | 用已撤销 Token 调用接口 | ✅ 返回 401 AUTH_REQUIRED |
| AC-T5 | 数据库查 `api_tokens` 表 | ✅ 仅有 `token_hash`，无明文列 |

### 6.2 插件配置
| 用例 | 步骤 | 预期 |
|---|---|---|
| AC-C1 | 全新安装插件打开 popup | ✅ 同步开关默认开启 |
| AC-C2 | 未填 Token 时查看 popup | ✅ 显著提示需配置 Token |
| AC-C3 | 填入有效 Token 保存后重开 popup | ✅ 状态显示「已连接」，Token 持久化 |
| AC-C4 | 填入无效 Token 后触发一次收藏 | ✅ 状态显示「Token 无效」 |

### 6.3 被动同步
| 用例 | 步骤 | 预期 |
|---|---|---|
| AC-P1 | 开关开启，按 Ctrl+D 收藏网页 A | ✅ 平台个人库新增 A，位于未分类，scope=USER |
| AC-P2 | 关闭开关，原生收藏网页 B | ✅ 平台不出现 B |
| AC-P3 | 在浏览器新建一个书签文件夹 | ✅ 不推送（无 URL 被跳过） |
| AC-P4 | 配置 Token 前已有的浏览器历史书签 | ✅ 不被导入平台 |
| AC-P5 | 原生收藏一个平台已存在的 URL | ✅ 平台不新增重复，无报错 |

### 6.4 主动收藏
| 用例 | 步骤 | 预期 |
|---|---|---|
| AC-A1 | 浏览网页 C，popup 点「收藏当前页」 | ✅ 提示「已收藏」，平台新增 C 于未分类 |
| AC-A2 | 对同一网页 C 再次点收藏 | ✅ 提示「已在书签库」，平台不新增 |
| AC-A3 | 收藏后检查平台书签字段 | ✅ 标题/URL/favicon 正确，scope=USER，无标签 |

### 6.5 去重一致性
| 用例 | 步骤 | 预期 |
|---|---|---|
| AC-D1 | 同一 URL 分别用主动、被动各收藏一次 | ✅ 平台仅一条记录 |
| AC-D2 | URL 带 ?utm 参数与不带版本分别收藏 | ✅ 按平台 normalizeUrl 规则判定是否同一条（与平台一致） |

### 6.6 失败与重试
| 用例 | 步骤 | 预期 |
|---|---|---|
| AC-F1 | 断网状态下主动收藏 | ✅ 提示稍后重试，载荷入队 |
| AC-F2 | 恢复网络后打开 popup | ✅ 队列自动 flush，书签成功进入平台，角标清除 |
| AC-F3 | 平台接口持续 500 | ✅ 单条重试至上限后标记失败，角标显示数量 |
| AC-F4 | Token 无效时收藏 | ✅ 不进入重试队列，直接提示检查 Token |

### 6.7 安全
| 用例 | 步骤 | 预期 |
|---|---|---|
| AC-S1 | 不带 Authorization 调用接口 | ✅ 401 |
| AC-S2 | 抓包确认请求未携带平台会话 Cookie | ✅ 仅 Bearer 头 |
| AC-S3 | 接口仅允许 https 平台域名（host_permissions） | ✅ http/其他域被浏览器阻止 |

## 7. 非功能验收
| 维度 | 要求 |
|---|---|
| 性能 | 主动收藏从点击到提示 ≤ 2s（正常网络） |
| 兼容 | Chrome 最新稳定版、Edge 最新稳定版可加载运行 |
| 可靠 | service worker 休眠后被书签事件唤醒仍能推送 |
| 隐私 | 插件仅请求 bookmarks/storage/activeTab/tabs/平台 host 权限 |
| 可维护 | 代码遵循 ai-toolkit/rules/coding 规范 |

## 9. 风险与待确认
| 项 | 说明 | 处置 |
|---|---|---|
| favicon 缺失 | 被动同步 `onCreated` 不带 favicon | v1 留空，标注 v2 增强 |
| 历史书签 | 用户期望旧书签也同步 | v1 明确不做，纳入 v2 候选 |
| 限流 | v1 仅记录不强制 | v2 接入正式限流 |
| 扩展 ID 固定 | CORS 精确放行需固定扩展 ID | 发布到商店后回填 `EXTENSION_ALLOWED_ORIGIN` |
